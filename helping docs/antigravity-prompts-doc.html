1. Advanced Chunking Strategies
Implement 8 different chunking strategies with benchmarking and testing capabilities.
Difficulty: Medium Time: 2-3 hours Priority: High
Task: Implement a comprehensive document chunking system for my RAG chatbot Requirements: 1. Create a Python module 'chunking_strategies.py' with these functions: - fixed_size_chunker(text, size=500, overlap=50) - sentence_based_chunker(text, sentences_per_chunk=5) - paragraph_based_chunker(text, min_length=100) - section_heading_chunker(text) # detects markdown/HTML headings - recursive_chunker(text, max_size=1000) - semantic_chunker(text, embedding_model) # using sentence transformers - sliding_window_chunker(text, window_size=400, stride=200) - metadata_aware_chunker(text, metadata_dict) 2. Create a ChunkingStrategy enum and factory pattern 3. Add comprehensive unit tests for each strategy 4. Include benchmarking scripts to compare strategies 5. Add documentation with use case recommendations File formats to support: PDF, DOCX, TXT, CSV, Excel Use libraries: langchain, PyPDF2, python-docx, pandas, sentence-transformers Verify the implementation works with sample documents from multiple domains (HR policies, medical protocols, legal contracts).
2. Multi-Document Query Handler
Build a system that retrieves and synthesizes information across multiple documents and file types.
Difficulty: Hard Time: 4-5 hours Priority: High
Task: Build a multi-document query system that can retrieve and synthesize information across different file types Requirements: 1. Create 'multi_doc_retriever.py' with: - DocumentLoader class supporting PDF, DOCX, Excel, CSV, TXT - VectorStore integration (Pinecone/ChromaDB/FAISS) - CrossDocumentRetriever that can: * Query multiple documents simultaneously * Rank results across documents * Attribute sources with document name + page/section * Handle conflicting information from different sources 2. Implement query routing based on: - Query intent classification (comparison, factual, procedural) - Document type detection - Metadata filtering (date, department, version) 3. Create response synthesis pipeline that: - Merges information from multiple chunks - Maintains source citations - Handles contradictions gracefully 4. Add caching layer for frequently asked questions 5. Test with: - HR policies (3+ documents) - Medical protocols (5+ documents) - Legal contracts (mixed formats) Include example queries like "Compare leave policies across employee levels" and "What are common requirements across all vendor contracts?"

Difficulty: Medium Time: 3-4 hours Priority: Medium

Create dynamic system prompt generator per persona 3. Implement document access control based on role 4. Add response formatting templates per persona type 5. Build persona-switching mechanism in conversation 6. Create 'chatbot_interface.py' with: - Streamlit UI with persona selector - Document upload interface (multi-file drag-drop) - Chat history with source citations - Query intent detection and clarification 7. Test scenarios: - Employee asking HR bot about leave policy - Doctor querying medical protocol bot - Sales rep using product knowledge bot Include confidence scoring and "I don't know" responses when information isn't in documents.

Implement SQL-like query translator: - Convert natural language to pandas operations - Handle aggregations (sum, average, count, group by) - Support filtering and sorting operations - Join operations across multiple files 3. Build hybrid retrieval: - Vector search for unstructured content - Direct table lookup for structured queries - Combined results for complex questions 4. Create visualization capabilities: - Auto-generate charts for numerical queries - Table formatting for comparison queries 5. Test scenarios: - "What's the average salary by department?" (from Excel) - "Show all employees with >10 days leave balance" (CSV) - "Compare Q3 vs Q4 sales across regions" (multiple sheets) Support file formats: XLSX, XLS, CSV, TSV Include error handling for malformed data
